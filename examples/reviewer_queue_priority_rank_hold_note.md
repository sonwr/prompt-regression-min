# Reviewer queue priority-rank hold note

Use this note when the exposed priority-rank winner exists, but the lead is too narrow to present as a confident next-focus handoff.

## When to use it

Use a hold note when all of these are true:

1. a queue still ranks first,
2. the margin over the runner-up is visible but small,
3. posting a hard winner would overstate confidence,
4. you still want a deterministic reviewer update.

## Copy-ready note

- `Current top queue: {winner}. Hold a hard next-focus call until the narrow lead over {runner_up} survives the next batch.`
- `Priority rank currently favors {winner}, but the margin over {runner_up} is still narrow enough that follow-up data should land before escalation.`

## Checklist

- name the current winner,
- name the runner-up,
- describe the lead as narrow,
- state the hold reason,
- say what should happen next.
