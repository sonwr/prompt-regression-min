# Reviewer queue priority-rank recheck

Use this when `priority_rank` is already exposed in JSON or PR-comment output, but you want one last recheck before posting a human handoff.

## Fast recheck order

1. Confirm the selected queue still has the best `priority_rank`.
2. Check whether the lead is wide enough to omit the runner-up.
3. If the lead is narrow, mention the runner-up explicitly.
4. Keep the final note to one sentence.

## Copy-ready line

`Next reviewer focus: <queue> (rank #<rank>) — <reason>; runner-up: <queue or n/a>.`

## When to mention the runner-up

- tied rank,
- narrow margin,
- or close queue-share / source-case-rate evidence.
