# Reviewer queue priority-rank tie card

Use this card when the reviewer queue exposes multiple follow-up groups with the same priority rank.

## Fast tie-break order

1. Prefer the group with the higher queue share.
2. If queue share ties, prefer the group with the higher active-case rate.
3. If active-case rate also ties, prefer the group with the higher source-case rate.
4. If all rates tie, keep the tie explicit in the handoff note.

## Copy-ready note

`Priority-rank tie: <group A> vs <group B>; keep <selected group> first because it leads on <queue share|active-case rate|source-case rate>.`
