# Reviewer queue bundle scope quick note

Use this note when you need one short reviewer update that keeps the selected queue tied to its shared artifact bundle.

## Template

`Reviewer queue: <queue>; bundle: <json/md/html basename>; reason: <why this queue stays next>.`

## Minimum proof

- name the winning reviewer queue
- name the shared report bundle or basename
- say why the queue stays next without reopening the full summary

## Good fit

Use this after the queue winner is already known and the remaining work is reviewer routing, not deeper regression analysis.
