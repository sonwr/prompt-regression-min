# Reviewer queue next-focus tie-break card

Use this card when multiple reviewer queues look close and you need one short rule for the handoff sentence.

## Order of preference

1. Higher priority label (`P1` beats `P2`)
2. Larger queued case count
3. Higher queue share
4. Higher active-case rate
5. Higher source-case rate
6. Stable alphabetical key as the final deterministic fallback

## Copy-ready sentence

`Next reviewer focus: <queue-key> because it wins the tie on priority, case count, and queue share.`

## If the tie is still effectively even

Say so directly:

`Next reviewer focus remains shared; queues are tied on priority and volume, so keep both visible in the handoff.`
