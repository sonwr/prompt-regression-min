# Reviewer queue priority rank replay note

Use this when you need a short human note after the tool has already ranked the queue.

## Minimal replay note

- Winner: copy `next_focus_label` and `next_focus_priority_label`.
- Margin: copy `next_focus_advantage_summary`.
- Runner-up: copy `runner_up_summary` if present.
- Scope: mention `queue_mix_summary` when the queue is spread across multiple buckets.

## Example

`Review regressions first; it leads the queue with the highest priority rank and a clear case-count advantage over formatting drift.`
