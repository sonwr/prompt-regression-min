# Reviewer queue top-two checklist

Use this note when the reviewer queue has a clear next focus but you still want a fast second-place comparison before posting a summary.

## Quick comparison

Check these in order:

1. **Priority rank** — confirm the next focus still wins by queue priority.
2. **Case count** — compare how many cases sit in the top queue vs the runner-up.
3. **Queue share** — confirm the lead is still meaningful, not just technically first.
4. **Source-case rate** — keep the overall dataset footprint visible.
5. **Human handoff** — make sure the posted note names both the next focus and the runner-up when the gap is small.

## Copy-ready sentence

```text
Reviewer queue top-two: fix regressions first, then watch unchanged fails if the regression lane clears.
```

## When to use it

- PR comment review before merge
- Summary markdown final pass
- Tied or near-tied reviewer queue situations
