# reviewer_queue_priority_margin_audit

Use this note when the reviewer queue has a winner, but the lead feels narrow.

## Fast audit

Check these fields together before posting a human handoff:

1. `next_focus_key`
2. `next_focus_priority_label`
3. `next_focus_advantage_summary`
4. `runner_up_summary`

## Decision rule

- If the selected queue leads on case count and queue share, keep the next-focus call.
- If case count ties but source-case rate or queue share is meaningfully better, explain that margin in one sentence.
- If every margin is effectively tied, post the tie summary instead of pretending the lead is decisive.

## One-line handoff pattern

`Next reviewer focus: <queue> because it still leads on queued follow-up and source-case rate.`
