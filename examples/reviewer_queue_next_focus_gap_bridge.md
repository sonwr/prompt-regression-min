# Reviewer Queue Next-Focus Gap Bridge

Use this note when the selected reviewer queue still leads, but the lead is small enough that you want a safer human handoff.

## When to use it

- `next_focus_priority_rank` is already exposed.
- The winner is real, but the runner-up still feels close.
- You want one sentence that keeps the lead visible without hiding the follow-up queue.

## Output pattern

`Next reviewer focus: <winner> remains first, but the lead is narrow — keep <runner-up> visible as the immediate follow-up.`

## Fill-in checklist

1. Name the exposed winner label.
2. Mention that the lead is narrow only when the queue-share or case-count gap is small.
3. Name the runner-up label.
4. Do not invent urgency that is not present in the summary payload.

## Example

`Next reviewer focus: policy remains first, but the lead is narrow — keep style visible as the immediate follow-up.`
