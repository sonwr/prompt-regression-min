# Reviewer queue priority-rank gap summary

Use this note when the top reviewer queue already has the visible priority-rank lead, but you still want to acknowledge that the gap is narrow.

## Copy-ready line

```text
Priority-rank lead: <winner> stays first, but the gap over <runner-up> is narrow enough that the next review update should keep both queues visible.
```

## When to use it

- The next-focus winner is clear.
- The runner-up still has enough queue share or source-case impact to matter.
- You want a one-sentence handoff instead of a longer comparison block.

## Example

```text
Priority-rank lead: policy-review stays first, but the gap over formatting-review is narrow enough that the next review update should keep both queues visible.
```
