# Reviewer queue next-focus advantage audit

Use this short audit when the summary already names a `next_focus_group`, but you want to sanity-check whether the callout is actually persuasive.

## Fast audit

1. Check whether the selected queue leads by case count, queue share, or source-case rate.
2. Check whether that lead is unique or part of a tie.
3. Check whether the handoff sentence explains the lead in human language.
4. Check whether the runner-up is close enough that reviewers should mention the tie pressure.
5. Check whether the chosen queue still matches the repo's stated review priority.

## Ready-to-post rule

Post the next-focus callout when the advantage summary tells a reviewer:

- why this queue comes first,
- how strong the lead is,
- and whether a runner-up deserves mention.

If those answers are missing, regenerate or rewrite the handoff before posting it to PR review.
