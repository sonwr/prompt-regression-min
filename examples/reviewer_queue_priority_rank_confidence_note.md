# Reviewer queue priority-rank confidence note

Use this note when the exposed priority rank is technically correct, but the lead still feels narrow enough that a reviewer may want a confidence qualifier.

## When it helps

Use it when:

- one queue is ranked first,
- the runner-up is still visible in the summary,
- and you want a one-line note that keeps the handoff honest.

## Copy-ready note

`Priority rank still favors <queue>, but the lead is narrow enough to keep <runner-up> visible in the follow-up note.`

## Example

`Priority rank still favors fix_regressions, but the lead is narrow enough to keep review_filtered visible in the follow-up note.`

## Fast check

If the note hides the runner-up entirely, it is not a confidence note anymore; it is just a winner declaration.
