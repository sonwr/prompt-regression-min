# Reviewer queue priority-rank winner note

Use this pattern when the summary already exposes `priority_rank` and you only need a compact human handoff.

## One-line template

`Next reviewer focus: <queue> (priority rank <rank>, share <share>, source-case rate <rate>).`

## When to use it

Use this note when:

- one queue already wins on exposed priority rank,
- the lead is clear enough that you do not need a full tie note,
- you still want the handoff to mention visible evidence.

## Keep it honest

Add a margin or runner-up qualifier when the lead is narrow.
Prefer `reviewer_queue_priority_rank_margin_summary.md` if the winner is technically ahead but still close.
