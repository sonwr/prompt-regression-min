# Reviewer queue priority-rank scope release gate

Use this note when a release-facing handoff needs one short queue winner plus one scope guard.

## Rule

If `priority_rank=1` is clear, name the winning queue first.
If the lead is narrow, keep the runner-up visible in the same sentence.
If the lead is not credible, hold the handoff instead of overstating certainty.

## Copy pattern

- Winner-only: `Next reviewer focus: <label> (P1). Shared report bundle stays reviewable.`
- Narrow lead: `Next reviewer focus: <label> (P1), with <runner-up> still close behind. Shared report bundle stays reviewable.`
- Hold: `No single reviewer focus is decisive yet; keep the shared report bundle open before routing a winner.`

## Minimum check

Before posting, confirm:

- the winning label comes from the saved summary artifact,
- any runner-up note matches the same artifact,
- the sentence does not claim stronger certainty than the queue data supports.
