# Reviewer queue priority-rank ready post

Use this snippet when the priority-ranked next focus is already decided and you want one short reviewer-facing update.

Template:

`Next focus: <winner> (<priority label>) -> <case ids>; runner-up: <runner up or none>; shared bundle: <summary.json|summary.md|summary.html>.`

Checklist:

- Keep the winner key and priority label together.
- Keep the runner-up visible when the lead is narrow.
- Name the shared artifact bundle so the reviewer can reopen evidence without guessing filenames.
