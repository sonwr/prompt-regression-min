# reviewer_queue_markdown_bundle_note

Use this when the active reviewer queue is already clear and you want a compact note that points reviewers to the generated markdown summary first.

## One-line rule

Open `summary.md` before reopening raw case JSON when the next review needs the highest-signal regression overview first.

## Why it helps

- keeps the first reopen step lightweight,
- preserves the queue decision in a human-readable artifact,
- and gives reviewers one stable markdown file to quote in PR threads.
