# Reviewer queue priority-rank scope note

Use this note when the reviewer queue exposes a winner, but the lead is still narrow enough that the handoff should stay scoped.

## When to use it
- `priority_rank` already identifies the current winner.
- The winner still has a close runner-up or narrow margin.
- You want one sentence that keeps the next action honest instead of sounding fully settled.

## One-line pattern
> Queue `<winner>` is still the next reviewer focus, but keep the handoff scoped because the visible rank lead over `<runner-up>` is narrow.

## What to verify first
- the exposed winner still matches `priority_rank = 1`
- the runner-up is still the closest competing queue
- the margin cue in JSON/markdown/PR-comment outputs still agrees
