# reviewer queue html report bundle check

Use this quick check before posting an HTML-ready reviewer queue update.

1. Confirm the summary JSON and summary Markdown point to the same queue winner.
2. Confirm the HTML report mentions the same next-focus queue and source-case impact.
3. Confirm the PR comment or handoff line names the exact artifact path reviewers should open first.
4. If the HTML report is missing or stale, fall back to the JSON/Markdown bundle and do not oversell HTML readiness.
