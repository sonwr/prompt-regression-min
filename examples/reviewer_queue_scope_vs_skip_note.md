# Reviewer queue scope vs skip note

Use this note when reviewers need one short status line that distinguishes scope filtering from rerun debt.

- **Filtered-out scope** means the case was intentionally excluded by the current selector.
- **Skipped case** means the case stayed in scope but still needs execution or evidence.
- Mention both counts together so a quiet regression queue does not hide skipped follow-up work.
- If both are non-zero, list scope first and rerun debt second.
