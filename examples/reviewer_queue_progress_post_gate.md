# Reviewer queue progress post gate

Use this note before posting a very short reviewer-queue update.

Post only when all three remain true:

- one queue is still the active next focus,
- the matching JSON/Markdown/HTML bundle still exists,
- and the saved artifacts explain *why* that queue is next without reopening raw case rows.

If one of those conditions is false, reopen the bundle or re-run the command before posting the status line.
