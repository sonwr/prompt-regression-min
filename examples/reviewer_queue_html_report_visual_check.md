# Reviewer queue HTML report visual check

Use this checklist when the regression summary or reviewer-queue HTML report layout changes.

## Quick pass

1. Generate the HTML artifact from a fixture with at least two queues.
2. Confirm the winning queue, runner-up, and confidence or margin cue are visible above the fold.
3. Verify any queue-share or source-case labels match the JSON summary for the same run.
4. Check that narrow-lead wording stays scoped and does not overstate confidence.
5. Reopen the file once after regeneration to confirm the artifact is self-contained and renders without external assets.
