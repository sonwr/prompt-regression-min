# reviewer_queue_priority_rank_margin_gate

Use this when `reviewer_queue.priority_rank` already exposes a winner, but the lead still looks narrow.

## Quick gate

Post the priority-rank winner only when all three are true:

1. The winner also leads in raw case count.
2. The runner-up does not tie on queue share after rounding to the same visible precision.
3. The handoff sentence still names the runner-up when the lead is one case.

## Copy-ready note

`Next reviewer focus: <winner> remains first, but the lead is narrow, so keep <runner-up> visible in the same handoff.`
