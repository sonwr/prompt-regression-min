# Reviewer queue scope status rule

Use this note when the next reviewer focus is known, but the handoff should stay intentionally narrow.

## Goal

A scope-safe status line should answer four questions in one sentence:

- which queue is next,
- why it is next,
- how strong the lead is,
- what reviewers should do next.

## Template

`Next reviewer focus: <queue label> (<priority cue>); lead is <clear|narrow|tied> based on <queue share/source-case rate>, so review <specific follow-up>.`

## Examples

- `Next reviewer focus: Safety policy drift (clear lead); lead is clear based on queue share, so review the policy cases first.`
- `Next reviewer focus: Billing edge cases (narrow lead); lead is narrow based on source-case rate, so confirm runner-up overlap before handoff.`
- `Next reviewer focus: Tone regressions (tied); lead is tied based on queue share, so keep both tied queues visible in the owner note.`

## Posting check

Keep the line to one sentence and avoid implying that the whole queue has already been resolved.
