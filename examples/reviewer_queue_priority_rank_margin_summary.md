# reviewer queue priority-rank margin summary

Use this short summary when the next reviewer queue already has a priority rank and you want a slightly fuller handoff than the single-line margin note.

## Template

`Priority rank: P{priority_rank} {queue_label}. Lead: {margin_cases} case(s) / {margin_queue_share}% queue share over the runner-up, so keep {runner_up_queue_label} visible as the fallback follow-up.`

## Example

`Priority rank: P1 fix regressions. Lead: 1 case(s) / 8.33% queue share over the runner-up, so keep policy review visible as the fallback follow-up.`

## When to use it

- The winning queue is still the right next focus, but the gap is slim.
- You want one sentence that names both the winner and the fallback queue.
- A PR comment needs slightly more context than the shortest status line.
