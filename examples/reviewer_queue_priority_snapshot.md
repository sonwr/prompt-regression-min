# Reviewer queue priority snapshot

Use this quick note when you need to explain the current reviewer queue in one screen.

## Snapshot recipe

1. Run the normal summary command for the queue.
2. Capture the dominant focus label and queue share.
3. Note the next-focus group and its margin versus the runner-up.
4. Add the owner or follow-up lane only when it changes the action.

## Compact wording pattern

- dominant focus: what is taking the most attention now
- next focus: what moves up if the dominant queue clears
- margin: how far apart the top two groups are
- action: review now, hand off, or wait for more cases

## Why this helps

A stable snapshot wording makes CI comments, markdown handoffs, and stdout summaries easier to compare across runs.
