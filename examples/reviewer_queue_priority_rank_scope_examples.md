# reviewer queue priority-rank scope examples

Use these examples when the priority-rank winner is real, but the public handoff should stay intentionally narrow.

## Winner-only scope note

`P1 fix regressions remains the next reviewer focus for this shard; runner-up queues stay unchanged.`

## Narrow-lead scope note

`P1 fix regressions stays first, but the lead is narrow enough that watch unchanged fails should stay visible in the same reviewer thread.`

## Shared-focus scope note

`Priority rank is tied at the top, so keep the first listed queue as the working lane while preserving the tie in the public note.`

## When to use this file

Use these lines after the JSON/markdown summary already exposed `next_focus`, `runner_up`, or tie metadata and you only need the final human-facing wording.
