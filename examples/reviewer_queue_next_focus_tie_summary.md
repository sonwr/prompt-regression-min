# reviewer queue next-focus tie summary

Use this note when `reviewer_queue.next_focus_tie_mode` is `tied` and you want a short reviewer handoff that keeps every same-sized first lane visible.

Template:

- Tie summary: `{next_focus_tie_summary}`
- Primary handoff: `{next_focus_handoff_summary}`
- Runner-up handoff: `{runner_up_handoff_summary}`

Quick rule:

- keep the deterministic first lane from the priority order
- keep the tied-lane summary in the same note
- avoid pretending the first lane is a unique winner when queue sizes are equal
