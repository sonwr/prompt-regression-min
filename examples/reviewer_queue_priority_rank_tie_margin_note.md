# Reviewer queue priority-rank tie margin note

Use this note when the exposed priority rank is tied and you still need a human-readable handoff.

## Template

`Priority rank is tied between <queue A> and <queue B>; keep <queue A> first only if queue share or source-case impact still gives it the clearer immediate review path.`

## Keep it honest

- Name the tied queues.
- Say what breaks the tie.
- Avoid sounding like there is a decisive winner if the lead is narrow.
