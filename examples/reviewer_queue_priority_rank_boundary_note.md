# Reviewer queue priority-rank boundary note

Use this note when the exposed priority rank is useful, but the handoff still needs a clear boundary before posting.

## What to include

- the queue that stays first,
- the runner-up that remains visible,
- the evidence that keeps the lead credible,
- and the condition that would trigger a recheck.

## Copy-ready pattern

```text
Keep <queue> as the next reviewer focus; <runner-up> stays visible, but the current lead still holds on <share/rate evidence>. Recheck if the next batch changes that gap.
```

## Quick audit

1. Does the note explain why the lead is real instead of just numerically first?
2. Does it keep the runner-up visible when the margin is narrow?
3. Does it name a concrete recheck trigger?
