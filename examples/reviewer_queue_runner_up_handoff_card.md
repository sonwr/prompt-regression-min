# reviewer_queue_runner_up_handoff_card

Use this note when the dominant reviewer queue is clear, but the runner-up still deserves visibility.

## Copy-ready template

`Primary follow-up remains <next_focus_label>; keep <runner_up_label> queued next because it still holds <runner_up_case_count> case(s) / <runner_up_queue_share> of the reviewer queue.`

## When it helps

- The next reviewer focus is unique, but the second queue is still large enough to mention.
- You want a short PR-comment sentence without reopening the JSON report.
- A maintainer wants to show sequencing (`now` vs `next`) in one line.
