# reviewer_queue_priority_rank_release_gate

Use this one-line handoff when the exposed priority-rank winner is strong enough to become the release gate focus.

Template:

`Release gate focus: <winner> remains P1, covering <queue_share>% of queued follow-up and <source_case_rate>% of source cases; keep the runner-up visible only if the margin is narrow.`

Example:

`Release gate focus: fix_regressions remains P1, covering 60.00% of queued follow-up and 30.00% of source cases; keep watch_unchanged_fails visible because the lead is still narrow.`
