# reviewer queue priority-rank post-ready check

Use this note right before posting a reviewer handoff that already exposes the priority-rank winner.

## Fast audit

Confirm the handoff still shows:

1. the winning queue key or label,
2. the visible priority rank,
3. whether the lead is unique or tied,
4. the next action for the reviewer.

## Copy-ready prompt

> Post only if the priority-rank winner, tie context, and next reviewer action still match the current summary payload.

## Why it matters

Priority-rank fields are concise, but they are easy to overstate when the lead is narrow. This short check keeps reviewer handoffs honest and paste-ready.
