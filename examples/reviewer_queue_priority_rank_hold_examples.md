# Reviewer Queue Priority-Rank Hold Examples

Use this note when a visible priority-rank winner should still stay on hold.

- Hold example: `fix_regressions` leads, but the gap is only one case and the runner-up affects the same release lane.
- Hold example: the queue winner is visible, but the linked artifact bundle still needs a refreshed HTML report.
- Hold example: the rank is stable enough for triage, but not strong enough for a decisive owner handoff.

Short handoff line:

`Priority rank currently favors <winner>, but keep this as a hold until the shared artifact bundle and runner-up gap are rechecked.`
