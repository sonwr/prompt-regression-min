# CLI Summary Owner Status One-Line Note

Keep reviewer-facing summary bundles short enough to skim without losing ownership or recommendation context.

- Put the owner, honest status, and saved summary bundle path in one short line when posting a follow-up.
- Keep the line grounded in validated output (`--summary-json`, `--summary-markdown`, or `--summary-pr-comment`), not hand-written guesses.
- Prefer one bundle per rerun so the recommendation, queue focus, and saved artifacts stay aligned.
