# Queue focus / owner / status bridge note

Use this note when one CLI summary artifact needs to answer three questions at once:

- what queue or scope is being reviewed,
- who owns the next action,
- what the current status really is.

Good summary wording stays short enough for PR comments and saved report bundles while still making the next review action obvious.

Recommended order:

1. queue focus,
2. owner,
3. status,
4. next review action if needed.
