# CLI summary five-repo Korean short-line note

Keep the cron-facing delivery brutally short: one Korean status line per repo, exactly five lines total.

Each line should keep the same order:

1. repo name,
2. change state,
3. validation pass/fail,
4. commit/push result or short hold reason.

This note exists so saved summary bundles can stay machine-friendly while the user-facing report stays Korean, compact, and honest about validation gates.
